2. Innovation: The Dynamic Style Registry (Evolutionary Schema)

SOTA Concept: Schema-Guided Reasoning (SGR)

Unlike static prompts, our Dynamic Style Registry evolves as the agent progresses through different sections of a long document.

Incremental Patching (RELAY)

If the model encounters a new visual element (e.g., a "Warning Box" or "Code Snippet" style not in the seed pages), it generates a Style Patch. This patch updates the Registry without restarting the process.

Style Conformity Scoring

Every page receives a style_conformity score (0-1). A score < 0.7 signals that the current document section has shifted style, triggering an autonomous re-calibration of heading weights.

3. The Triplet Context Logic

Spatio-Temporal Reasoning: By receiving [Previous, Target, Next] pages, the AI resolves the "Stitch Problem".
  • Left-Context (N-1): Resolves if Target starts with a sentence fragment.
  • Right-Context (N+1): Resolves if a table in Target is continued or finalized.

The Triplet Prompt Strategy

[SYSTEM_SCHEMA]
Use the current Dynamic Style Registry: {{registry_json}}

[INPUT_BUFFER]
- IMAGE_PREV: Reference for paragraph joining.
- IMAGE_TARGET: The PRIMARY EXTRACTION TARGET.
- IMAGE_NEXT: Reference for element finalization.

[RULES]
1. OUTPUT ONLY Target content.
2. If Target text is a continuation of Prev, do not repeat the header.
3. If Target ends with an incomplete Table, mark in JSON: "status": "continued".
